## 페이지 1: 대시보드 (Dashboard)

### 왜 필요한가
담당자가 출근 후 가장 먼저 보는 화면. 전날의 후원 현황, 미납 경보, 오늘 해야 할 일을 한 눈에 파악해야 한다. MRM에는 이 기능이 약하고, Bloomerang은 Donor Retention 중심 대시보드를 강점으로 내세운다.

### 화면 구성

```
┌──────────────────────────────────────────────────────────┐
│  [오늘 날짜]  [단체명]           [알림 3]  [계정]        │
├──────────┬──────────┬──────────┬────────────────────────┤
│ 이번달   │ 이번달   │ 이번달   │ 이번달                 │
│ 신규후원자│ 모금액   │ 해지수   │ 미납건수               │
│  +15명   │ 2,340만  │ 3명      │ 12건                   │
├──────────┴──────────┴──────────┴────────────────────────┤
│  [월별 후원금 추이 차트 - 최근 12개월 Bar Chart]         │
├──────────────────────┬──────────────────────────────────┤
│  오늘의 할 일         │  최근 결제 오류 현황              │
│  ─────────────────── │  ────────────────────────────── │
│  ▪ CMS 출금 확인 (3) │  잔액부족: 5건  해지계좌: 2건    │
│  ▪ 미납 알림 발송 (7)│  카드만료: 1건                   │
│  ▪ 영수증 발급 대기  │  [바로가기 →]                    │
│    (12건)            │                                  │
├──────────────────────┴──────────────────────────────────┤
│  최근 신규 후원자 (오늘)                  [더보기]       │
│  홍길동 | 정기 3만원 | 카카오페이 | 10분 전             │
│  이순신 | 일시 5만원 | 카드       | 32분 전             │
└──────────────────────────────────────────────────────────┘
```

### 주요 위젯 및 KPI

| 위젯 | 지표 | 설명 |
|------|------|------|
| 모금 현황 | 이번달 모금액 / 목표 대비 % | 캠페인별 합산 |
| 후원자 현황 | 신규/유지/해지 건수 | 전월 대비 |
| 미납 경보 | 미납 건수 / 금액 | 클릭 시 미납 목록으로 이동 |
| 결제 오류 | 오류 유형별 건수 | CMS, 카드 구분 |
| 할 일 목록 | 오늘/이번주 처리 항목 | 클릭 시 해당 기능으로 이동 |
| 후원자 이탈 경보 | 3개월 이상 미납 후원자 | Bloomerang 벤치마킹 |

### Workflow

```
[로그인] → [대시보드 로드]
         → 각 위젯별 API 병렬 호출
         → 이탈 위험 후원자 자동 계산 (미납 3개월 이상)
         → 오늘 스케줄 CMS 출금 건수 표시
         → [클릭] → 해당 상세 페이지 이동
```

## 페이지 2: 후원자 목록 (CRM 목록 화면)

### 왜 필요한가
MRM 스크린샷에서 확인된 핵심 화면. 모든 후원자를 조회하고 필터링하며, 개별 후원자로 드릴다운할 수 있는 허브 역할이다. CMS 회원 / 카드 회원 / 웹 가입 회원이 여기서 통합되어야 한다.

### 화면 구성

```
┌──────────────────────────────────────────────────────────┐
│ [검색창: 이름/회원번호/연락처]  [그룹 ▼] [상태 ▼] [조회]│
│ [납부방법 ▼] [납부유형 ▼] [기부시작일 범위]  [엑셀 내보내기]│
├────┬─────────┬──────────┬──────────┬──────┬──────┬──────┤
│ □  │회원번호  │ 이름     │ 기부시작 │납부방│납부유│후원  │
│    │연락처    │ 이메일   │ 기부금액  │법    │형    │상태  │
├────┼─────────┼──────────┼──────────┼──────┼──────┼──────┤
│ □  │M001234  │ 홍길동   │ 2020-01  │CMS   │정기  │ 정상 │
│    │010-1234 │ h@g.com  │ 30,000원  │      │      │      │
├────┼─────────┼──────────┼──────────┼──────┼──────┼──────┤
│ □  │M001235  │ 이순신   │ 2019-06  │카드  │정기  │ 정상 │
│ ...│         │          │          │      │      │      │
├────┴─────────┴──────────┴──────────┴──────┴──────┴──────┤
│ 총 1,240명  | 정기납: 1,100 | 일시납: 140 | 미납: 45     │
└──────────────────────────────────────────────────────────┘
```

### 컬럼 정의

| 컬럼명 | 설명 | 정렬/필터 |
|--------|------|----------|
| 회원번호 | 시스템 자동 부여 | 검색 가능 |
| 이름 | 성명 (법인은 기관명) | 검색 가능 |
| 연락처 | 휴대폰 번호 | 검색 가능 |
| 기부시작일 | 최초 후원 시작 날짜 | 범위 필터 |
| 납부금액 | 현재 약정 금액 | 정렬 |
| 납부방법 | CMS/카드/계좌이체/지로 | 필터 |
| 납부유형 | 정기납/일시납 | 필터 |
| 후원상태 | 정상/정지/해지/대기 | 필터 |
| CMS 상태 | 정상/오류/해지 | 필터 |
| 최근 납부 | 마지막 납부일 | 정렬 |

### 주요 기능

- **다중 선택 후 일괄 처리**: SMS 발송, 그룹 이동, 상태 변경
- **컨텍스트 메뉴 (우클릭)**: 그룹변경 / 납부정보 다운로드 / 상담이력 / SMS 전송
- **하단 요약 바**: 선택된 건의 합계 금액, 인원수
- **하단 차트**: 선택 후원자의 월별 납부 추이 미니 차트
- **우측 미리보기 패널**: 선택 후원자 기본 정보 빠른 확인

### Workflow — 신규 후원자 등록

```
[+신규 등록 버튼] 
→ 후원자 상세 페이지 (등록 모드)
→ 기본정보 입력 (이름, 연락처, 이메일, 주소, 생년월일)
→ 후원 유형 선택 (정기/일시)
→ 금액 및 결제수단 입력
→ [저장] → 중복 회원 자동 검사
   → 중복 발견 시: "동일 연락처 회원 존재" 팝업 → 통합 or 신규
   → 중복 없음: DB 저장 → 목록으로 복귀
→ 자동 감사 SMS 발송 트리거
```

### 예외 처리

| 상황 | 처리 |
|------|------|
| 동일 연락처 중복 등록 | 팝업으로 기존 회원 표시, 통합 여부 확인 |
| 주민번호 불일치 | 영수증 발급 불가 경고 표시 |
| 필수항목 누락 | 인라인 유효성 검사 (저장 전) |
| 삭제 시도 | 납부 이력 있는 경우 삭제 불가, 비활성화만 가능 |

## 페이지 3: 후원자 상세 / 등록 / 수정

### 왜 필요한가
MRM 스크린샷 1번에서 확인된 핵심 화면. 단일 후원자의 모든 정보를 탭으로 구조화하여 관리하는 허브. 결제정보 변경, 납부이력 확인, 상담이력 기록이 한 화면에서 이루어진다.

### 화면 구성 (탭 구조)

```
┌──────────────────────────────────────────────────────────┐
│ 회원번호: M001234  [상태: 정상 ●]  [편집] [SMS] [이메일] │
├───────┬─────────┬──────────┬──────────────────────────── │
│기본정보│ 결제정보 │ 가족관계  │ 상담이력  │ 파일                │
├───────┴─────────┴──────────┴──────────────────────────── │
│                        [기본정보 탭 내용]                 │
│  이름: 홍길동          │ 생년월일: 1985-03-15             │
│  연락처: 010-1234-5678 │ 이메일: hong@email.com           │
│  주소: 서울시 강남구...│ 주소2: (선택)                    │
│  회원구분: ○일반 ○법인  │ 가입경로: 온라인-캠페인A        │
│  기부시작일: 2020-01-05│ 기부종료일: (계속 후원 중)       │
│  기부금액: 30,000원/월 │ 기금: 일반후원                   │
├──────────────────────────────────────────────────────────┤
│  [우측 사이드 패널: 납부 이력 타임라인]                   │
│  2026-04 ● 납부완료 30,000원                             │
│  2026-03 ● 납부완료 30,000원                             │
│  2026-02 ✕ 미납 (잔액부족)                               │
│  2026-01 ● 납부완료 30,000원                             │
│  [더보기]                                                 │
└──────────────────────────────────────────────────────────┘
```

### 탭별 상세 내용

#### 탭 1 — 기본정보

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 이름 (성명) | text | 필수 | |
| 회원구분 | select | 필수 | 개인/법인 |
| 회원상태 | select | 필수 | 정상/정지/해지/잠재 |
| 생년월일 / 주민등록번호 | date / masked | - | AES-256 암호화 저장 |
| 연락처 (휴대폰) | tel | 필수 | |
| 이메일 | email | 권장 | |
| 주소 | address | - | 우편번호 연동 |
| 가입경로 | select | - | 온라인/오프라인/소개 등 |
| 기부시작일 | date | 자동 | 최초 납부 완료일 |
| 기부금액 | number | - | 현재 약정 금액 |
| 기금 / 사업코드 | select | - | 기금 코드 테이블 참조 |
| 추천인 | member_search | - | 회원 검색 연결 |
| 메모 | textarea | - | 내부 메모 |
| 사용자 정의 필드 1~20 | 각종 타입 | - | 단체 설정에 따라 |

#### 탭 2 — 결제정보

| 필드 | 타입 | 설명 |
|------|------|------|
| 납부유형 | radio | 정기납 / 일시납 |
| 납부방법 | select | CMS / 신용카드 / 계좌이체 / 지로 |
| CMS 계좌번호 | masked | 암호화 저장 |
| 카드번호 | masked | 토큰화 저장 |
| 출금일 | select | 매월 N일 |
| 약정 목록 | table | 약정코드, 금액, 시작일, 상태 |
| [약정 추가] 버튼 | | 신규 약정 등록 |

#### 탭 3 — 가족관계

- 배우자, 자녀 등 연결 후원자 관리
- Salesforce NPSP의 Household 개념 적용
- 가족 합산 기부금 조회 가능

#### 탭 4 — 상담이력

```
[+새 상담 기록]
─────────────────────────────────────
2026-03-10  담당: 김관리자
유형: 전화 / 문의: 결제수단 변경 요청
내용: CMS → 카카오페이로 변경 희망
처리: 결제수단 변경 완료
─────────────────────────────────────
2025-12-20  담당: 이관리자
유형: 이메일 / 문의: 주소 변경
...
```

#### 탭 5 — 파일 첨부

- 파일 업로드 (드래그&드롭)
- 파일 유형: 문서, 이미지, 동영상, 스캔본, 녹취파일
- 모든 파일 암호화 저장
- 다운로드 이력 기록

### Workflow — 결제수단 변경

```
[결제정보 탭] → [편집 모드]
→ 납부방법 변경 (예: CMS → 카카오페이)
→ 새 결제수단 정보 입력
→ [저장]
→ 변경 이력 자동 기록 (변경 전/후/변경일/담당자)
→ 다음 출금일부터 새 결제수단으로 처리
→ 변경 완료 SMS 자동 발송 (선택)
```

### Workflow — 월마감 납부 처리

```
[매월 출금일 D+1]

[관리자 확인]
→ 미납 목록 확인 → SMS/알림톡 발송
→ 오류 건 확인 → 원인 분류 후 처리

[ERP 연동]
→ [ERP 전송] 버튼 클릭
→ API 호출: GET /api/v1/payments?incomeStatus=1
→ ERP에서 수입결의서 생성
→ API 호출: PATCH /api/v1/payments/{id}/status → 수입완료
→ 화면 수입상태 업데이트
```

| ERP 전송 | 수입대기 건 | API 일괄 호출 |

## 페이지 5: 미납 관리

### 왜 필요한가
미납은 단체의 재정에 직접 영향을 미친다. MRM 스크린샷 3, 8번에서 확인. 출금 오류 유형별로 대응 방법이 다르므로 원인 분류가 핵심이다.

### 화면 구성

```
┌──────────────────────────────────────────────────────────┐
│  [기간] [납부방법] [오류유형] [미납개월수 ≥]    [조회]   │
├──────────────────────────────────────────────────────────┤
│  요약 카드:                                               │
│  잔액부족: 28건 / 1,200만원  |  해지계좌: 5건 / 350만원  │
│  지급정지: 3건 / 150만원     |  카드만료: 9건 / 540만원  │
├────┬─────────┬──────┬──────┬──────┬──────┬─────────────┤
│ □  │회원번호  │이름  │미납월│연속  │금액  │오류유형     │
│    │         │     │ 수   │ 회수 │     │             │
├────┼─────────┼──────┼──────┼──────┼──────┼─────────────┤
│ □  │M001234  │홍길동│ 3   │  3   │90,000│잔액부족     │
│ □  │M001235  │이순신│ 1   │  1   │50,000│해지계좌     │
├────┴─────────┴──────┴──────┴──────┴──────┴─────────────┤
│  [선택 SMS 발송] [선택 일시정지] [선택 수입제외] [엑셀]  │
└──────────────────────────────────────────────────────────┘
```

### 오류 유형별 처리 정책

| 오류 유형 | 자동 처리 | 관리자 조치 |
|----------|---------|-----------|
| **잔액부족** | D+1, D+3, D+7 재시도 | 재시도 후 미납 SMS 발송 |
| **해지계좌** | 즉시 스케줄 정지 | 결제수단 변경 안내 SMS |
| **지급정지** | 즉시 스케줄 정지 | 개별 전화 확인 필요 |
| **카드만료** | 만료 전달 갱신 안내 발송 | 카드정보 업데이트 유도 |
| **한도초과** | D+1 재시도 | 재시도 후 미납 SMS |
| **연속 3회 실패** | 스케줄 자동 일시정지 | 상담 이력 생성 |

### Workflow — 미납 처리 자동화

```
[출금 실패 Webhook 수신]
→ 오류 유형 분류
→ 미납 레코드 생성
→ 자동 재시도 큐 등록 (잔액부족/한도초과)
→ 관리자 대시보드 알림

[자동 SMS 발송]
→ 출금 실패 D+0: "출금이 실패했습니다. 잔액을 확인해주세요."
→ 출금 실패 D+3: "재출금을 시도합니다. 잔액을 준비해주세요."
→ 최종 미납 확정 후: 재결제 링크 포함 SMS

[3개월 연속 미납]
→ 스케줄 자동 일시정지
→ 관리자 상담 알림 생성
→ 이탈 위험 후원자 대시보드에 표시
```

## 페이지 8: ERP 수입결의 연동

### 왜 필요한가
BF_IO 문서에서 확인된 ERP 수입 처리 프로세스를 시스템화. 도너스의 수동 API 연동 방식을 개선하여, 관리자가 버튼 클릭 또는 자동으로 수입결의를 처리한다.

### 8-1 MRM 납입 내역 조회 (BF_IO_001)

```
┌──────────────────────────────────────────────────────────┐
│  조회 기간: [____] ~ [____]  날짜기준: ○납입일 ○입금일  │
│  수입상태:  □수입대기(1) □수입진행(2) □수입완료(3)       │
│             □수입제외(4)            [API 조회] [엑셀]   │
├────────────────────────────────────────────────────────── │
│  API 상태: ● 정상 연결  마지막 조회: 2026-04-15 09:30   │
├────┬─────────┬──────┬───────┬──────┬──────┬─────────────┤
│ □  │납입코드  │회원명│모금상품│납입액│납입일│수입상태     │
├────┼─────────┼──────┼───────┼──────┼──────┼─────────────┤
│ □  │P26001   │홍길동│일반후원│30,000│04-25 │수입대기     │
│ □  │P26002   │이순신│아동후원│50,000│04-25 │수입대기     │
├────┴─────────┴──────┴───────┴──────┴──────┴─────────────┤
│  [선택 건 → 수입결의 등록]  [전체 → 수입결의 업로드]    │
└──────────────────────────────────────────────────────────┘
```

### 8-2 수입결의서 등록 (BF_IO_002)

**Workflow:**

```
[납입 내역 선택] → [수입결의 등록 버튼]
  ↓
수입결의서 입력 폼:
  - 결의일자: [자동: 오늘]
  - 수입과목: [기금 코드 → 회계 과목 자동 매핑]
  - 납입자명 / 납입금액 / 납입방법: [자동 채워짐]
  - 적요: [자동 생성 or 수정 가능]
  - 비고: [선택 입력]
  ↓
[저장] → ERP API 호출: PATCH /api/v1/payments/{id}/status
       → incomeStatus: 2 (수입진행)
       → ERP 내부 결의서 생성
  ↓
ERP 처리 완료 Callback 수신
  → incomeStatus: 3 (수입완료) 업데이트
  → 화면 갱신
```

### 8-3 수입결의서 일괄 업로드 (BF_IO_003)

```
[엑셀 템플릿 다운로드]
→ 파일 작성 (납입코드, 결의일자, 수입과목, 금액)
→ [파일 업로드 (xlsx)]
→ 유효성 검사:
   - 납입코드 존재 여부
   - 금액 일치 여부
   - 수입상태 = 수입대기 여부
→ 오류 있으면: 오류 목록 표시, 다운로드 가능
→ 오류 없으면: [확정 업로드] 버튼 활성화
→ 일괄 처리 → 완료 리포트
```

### 8-4 수입결의 확정 (BF_IO_005)

- 수입진행 상태의 결의서를 최종 확정
- 확정 후 수정 불가 (수정 필요 시 대체결의서 등록)
- 확정된 건 수입완료 상태로 변경

### 연동 오류 처리

| 오류 유형 | 처리 방법 |
|---------|---------|
| API 연결 실패 | 로컬 큐 저장 → 재시도 (최대 3회) → 실패 시 관리자 알림 |
| 상태값 충돌 (이미 완료) | 중복 처리 경고, 이력 기록 |
| 금액 불일치 | 불일치 경고 팝업, 관리자 확인 후 처리 |
| ERP 타임아웃 | 3초 대기 후 재시도, 결과 불명확 시 수동 확인 필요 팝업 |

## 전체 관리자 Workflow 요약

### Workflow 1: 월간 후원 수납 프로세스 (매월)

```
[1일] CMS 청구 파일 송신 (자동)
  ↓
[출금일 D-0] PG사 출금 실행
  ↓
[출금일 D+1] 결과 수신 (Webhook/파일)
  → 납부완료 건: 수입대기 상태 생성
  → 출금실패 건: 오류 유형 분류 + 관리자 알림
  ↓
[D+1~2] 관리자: 미납/오류 확인 → SMS 발송
  ↓
[D+3] 재시도 (잔액부족 건 자동)
  ↓
[D+5] 최종 미납 확정 → 이력 기록
  ↓
[D+5] ERP 수입결의 처리
  → 수입대기 건 일괄 조회
  → 수입결의서 등록 또는 업로드
  → 수입완료 상태 업데이트
  ↓
[월말] 기부금영수증 (월별 요청 시) 발급
  ↓
[익월 1일] 대시보드 월별 집계 갱신
```

### Workflow 2: 신규 후원자 온보딩

```
[후원자: 캠페인 페이지 접속]
  ↓
[결제 완료]
  → 납입 레코드 생성 (약정 + 첫 납입)
  → 감사 SMS/이메일 자동 발송
  → 후원증서 발급
  ↓
[관리자 시스템: 후원자 목록에 표시]
  → 그룹 자동 배정 (신규 후원자 그룹)
  → 온보딩 자동화 트리거 실행
     → D+7: 활동 안내 이메일
     → D+30: 첫 달 감사 메시지
  ↓
[CRM 등록 완료]
  → 상담이력 자동 생성 ("온라인 신규 가입")
```

### Workflow 3: 연말 기부금영수증 일괄 발급

```
[1월 초]
  ↓
[기부금영수증 관리] → 귀속연도: 작년 선택
  ↓
발급 대상 확인:
  - 주민번호 누락 회원 → 정보 요청 SMS 발송
  - 이메일 없는 회원 → 우편 발송 목록 분류
  ↓
[일괄 이메일 발송] → PDF 영수증 첨부
  ↓
[국세청 간소화 자료 생성] → 전산매체 파일 다운로드
  ↓
[홈택스 업로드] (관리자 직접 처리)
```
